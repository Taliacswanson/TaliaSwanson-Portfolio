Tuning

LASSO tuning (no cv)

# Recipe (SEX)
model_recipe <- recipe(Y ~ ., data = data) %>%
  step_dummy(all_nominal_predictors(), one_hot = TRUE) %>%
  step_zv(all_predictors())

# LASSO model (tunable)
lasso_spec <- linear_reg(penalty = tune(), mixture = 1) %>%
  set_engine("glmnet")

# LASSO workflow
lasso_workflow <- workflow() %>%
  add_model(lasso_spec) %>%
  add_recipe(model_recipe)

Defining your penalty grid

penalty_grid <- tibble(
  penalty = 10^seq(-5, 2, length.out = 50)
)
# Apparent resampling (full dataset)
resamples_app <- apparent(data)

# Tune LASSO over full dataset
lasso_tune_res <- tune_grid(
  lasso_workflow,
  resamples = resamples_app,
  grid = penalty_grid,
  metrics = metric_set(rmse)
)

As the LASSO penalty increases, the RMSE increases. This is expected because larger penalties shrink coefficients toward zero, reducing model flexibility and potentially leading to underfitting. With smaller penalties, the model behaves more like a standard linear regression.

However, this tuning is performed on the full dataset without cross-validation, meaning the model is evaluated on the same data used for training. This leads to overly optimistic performance estimates and increases the risk of overfitting.

# Random Forest model (tunable)
rf_spec <- rand_forest(
  mtry = tune(),
  min_n = tune(),
  trees = 300
) %>%
  set_engine("ranger", seed = 1234) %>%
  set_mode("regression")

# Random Forest workflow 
rf_workflow <- workflow() %>%
  add_model(rf_spec) %>%
  add_recipe(model_recipe)
# Define 5-fold CV repeated 5 times
cv_folds <- vfold_cv(data, v = 5, repeats = 5)

# Tune LASSO with CV
lasso_cv_res <- tune_grid(
  lasso_workflow,
  resamples = cv_folds,
  grid = penalty_grid,
  metrics = metric_set(rmse)
)

# Random Forest tuning with CV
rf_grid <- grid_regular(
  mtry(range = c(1, 7)),
  min_n(range = c(1, 21)),
  levels = 7
)

rf_cv_res <- tune_grid(
  rf_workflow,
  resamples = cv_folds,
  grid = rf_grid,
  metrics = metric_set(rmse)
)

autoplot(rf_cv_res)

When using cross-validation, the RMSE values are generally higher compared to tuning without cross-validation. This is because the model is evaluated on unseen data in each fold, providing a more realistic estimate of performance. Cross-validation helps prevent overfitting and gives a better indication of how the model will perform on new data.

Select best parameters

# Best LASSO penalty
best_lasso <- select_best(lasso_cv_res, metric = "rmse")

# Best RF parameters
best_rf <- select_best(rf_cv_res, metric = "rmse")

The best LASSO penalty minimizes RMSE across CV folds. Likewise, the best Random Forest parameters minimize RMSE and will be used to finalize each model .

Final models

# Final LASSO model
final_lasso <- finalize_workflow(
  lasso_workflow,
  best_lasso
) %>%
  fit(data)

# Final Random Forest model
final_rf <- finalize_workflow(
  rf_workflow,
  best_rf
) %>%
  fit(data)

Using the selected optimal parameter, we fit the final LASSO and Random Forest models to the full dataset. These models will be used for predictions and comparison.
